# Дообучение Mistral-7B методом QLoRA под стиль ответов

Учебный проект по дообучению `Mistral-7B-Instruct-v0.3` через LoRA на инструктивном датасете, с последующим сравнением базовой и дообученной модели.

## Структура репозитория

```
.
├── notebook.ipynb           # код: подготовка данных, обучение, оценка
├── train.jsonl              # финальный датасет (206 примеров)
├── lora_adapter/            # сохранённые веса LoRA-адаптера
├── training_log.json        # история loss по шагам
├── loss_curve.png           # график обучения
├── comparison_results.json  # сравнение база vs дообученная (10 вопросов)
├── metrics.json             # ROUGE и BERTScore
└── README.md
```

## 1. Данные

Датасет собран на основе [alpaca-cleaned](https://huggingface.co/datasets/yahma/alpaca-cleaned): часть примеров взята готовыми, часть дописана самостоятельно — бытовые и образовательные темы (питание, спорт, обучение, тайм-менеджмент, технологии), чтобы расширить тематическое разнообразие.

**Итоговый размер: 206 примеров**, формат JSONL, поля `instruction` и `output`.

Фильтрация перед финальной сборкой:
- удаление дубликатов по полю `instruction` (сравнение без учёта регистра);
- отсев примеров с пустым или слишком коротким ответом (< 20 символов).

После фильтрации дубликатов в датасете не осталось — все 206 пар уникальны.

## 2. Fine-tuning

**Базовая модель:** `mistralai/Mistral-7B-Instruct-v0.3`
**Метод:** QLoRA — 4-bit квантизация (NF4) через `BitsAndBytesConfig` + LoRA-адаптеры
**Железо:** Google Colab, GPU T4 (бесплатный тариф)

### Конфигурация LoRA

| Параметр | Значение |
|---|---|
| r | 8 |
| lora_alpha | 16 |
| lora_dropout | 0.05 |
| target_modules | q_proj, k_proj, v_proj, o_proj |
| bias | none |

### Гиперпараметры обучения

| Параметр | Значение |
|---|---|
| epochs | 3 |
| per_device_train_batch_size | 2 |
| gradient_accumulation_steps | 8 |
| learning rate | 2e-4 |
| optimizer | paged_adamw_8bit |
| precision | fp16 |

Сохранён только адаптер (`lora_adapter/`, ~27 МБ), не полная модель.

### Кривая обучения

![Loss curve](loss_curve.png)

Loss снизился с **~1.22** в начале обучения до **~0.83** к концу третьей эпохи — модель устойчиво обучалась на всём протяжении прогона.

**Первая попытка обучения была неудачной** и в репозиторий не включена: при `logging_steps=10` и эффективном батче 16 на тогдашних 198 примерах и 1 эпохе обучение укладывалось всего в 13 шагов — лог содержал фактически одну точку, и построить содержательный график не получилось. Для финальной версии уменьшен `logging_steps` до 5 и увеличено число эпох до 3 — это и дало кривую выше.

## 3. Оценка

Сравнение проводилось на **10 вопросах**, не входящих в обучающую выборку — по каждому из них сгенерированы два ответа: базовой моделью и моделью с подключённым LoRA-адаптером.

### Ограничение: ответы сокращены

Бесплатный T4 в Colab даёт ограниченную и нестабильную по времени GPU-память, а сессия имеет лимит по длительности. Чтобы не потерять уже обученный адаптер и не упереться в обрыв сессии посреди генерации, длина ответа при оценке была ограничена 64 токенами (`max_new_tokens=64`). Из-за этого часть ответов в `comparison_results.json` обрывается на полуслове — это видно в примерах ниже и помечено знаком «…».

Сами ответы при этом получены реальной генерацией модели, не дополнены и не переписаны вручную — обрезаны только по длине.

### Метрики

Так как отдельного эталонного ответа для каждого вопроса нет, метрики считались между ответами **базовой и дообученной модели** — они показывают, насколько изменился стиль и формулировки, а не фактическую правильность содержания.

| Метрика | Значение |
|---|---|
| ROUGE-1 | 0.100 |
| ROUGE-2 | 0.000 |
| ROUGE-L | 0.100 |
| BERTScore Precision | 0.767 |
| BERTScore Recall | 0.767 |
| BERTScore F1 | 0.767 |

ROUGE измеряет совпадение по словам и n-граммам — низкие значения здесь ожидаемы, потому что модель перефразирует мысль другими словами, а не повторяет формулировки базовой модели. ROUGE-2 = 0 означает, что совпадающих биграмм между ответами практически нет — оба варианта построены разными фразами.

BERTScore оценивает смысловую близость через эмбеддинги, а не точное совпадение слов. F1 = 0.767 говорит о заметном смысловом сходстве при том, что формулировки разные — то есть модели обсуждают одно и то же, но по-разному его формулируют. Это более информативный сигнал, чем ROUGE, для случая, когда нет цели точно воспроизвести текст, а есть цель изменить стиль.

## Выводы

**Что получилось:**
- Полный цикл от подготовки данных до оценки воспроизведён и реально запускался на бесплатном T4.
- Адаптер обучился со стабильным снижением loss (1.22 → 0.83) и сохранён отдельно от базовой модели.
- В ответах дообученной модели заметна разница в формулировках и структуре по сравнению с базовой — например, чаще встречаются нумерованные списки и более прямая, инструктивная подача.
- BERTScore показывает высокое смысловое сходство при низком ROUGE — это означает, что модель не копирует базовые ответы, а перефразирует их по-своему, сохраняя смысл.

**Что не получилось / ограничения:**
- 206 примеров и ранг LoRA r=8 — небольшой объём данных и низкий ранг для модели 7B; этого недостаточно для кардинального изменения стиля, только для частичного сдвига.
- Ответы при оценке ограничены 64 токенами из-за ограничений бесплатного T4 в Colab — часть текста обрывается, что не позволяет оценить связность модели на длинных ответах.
- Метрики считались относительно базовой модели, а не относительно эталонных ("золотых") ответов — это ограничивает интерпретацию: метрики показывают степень отличия, а не степень правильности.
- Первый прогон обучения был сделан с неудачными параметрами логирования и не дал пригодного графика — пришлось перенастраивать `logging_steps` и количество эпох и обучать заново.

**Что можно улучшить дальше:**
1. Увеличить датасет до 500+ примеров.
2. Повысить ранг LoRA до 16–32.
3. Сгенерировать полные, не обрезанные ответы (нужен GPU с более стабильным временем сессии или платный тир Colab).
4. Добавить LLM-as-a-judge оценку через API для более содержательного сравнения смысла.
5. Использовать отдельный eval-сплит с эталонными ответами для более честной метрики.

## Технологии

- Transformers, PEFT, BitsAndBytes
- rouge-score, bert-score
- Google Colab (T4 GPU, бесплатный тариф)

## Запуск

1. Открыть `notebook.ipynb` в Google Colab.
2. Выбрать среду выполнения с GPU (T4).
3. Загрузить `train.jsonl`.
4. Выполнить ячейки по порядку: установка библиотек → подготовка данных → обучение → построение графика → оценка.
5. Результат: адаптер в `lora_adapter/`, метрики в `metrics.json`, сравнение в `comparison_results.json`.
